# ADR 003: 결과 Model

## Status

아키텍처 의사결정에서 Accepted을(를) 기준으로 데이터 품질 검증, 워크플로우 자동화, 결과 해석 방법을 설명합니다.

## Context

아키텍처 의사결정에서 Legacy을(를) 기준으로 데이터 품질 검증, 워크플로우 자동화, 결과 해석 방법을 설명합니다.

## Decision

아키텍처 의사결정에서 ValidationRunResult, Make을(를) 다루는 항목입니다:

- 아키텍처 의사결정에서 `CheckResult`, CheckResult을(를) 기준으로 데이터 품질 검증, 워크플로우 자동화, 결과 해석 방법을 설명합니다.
- issue 증거 through `ValidationIssue`
- 런타임 실패 through `ExecutionIssue`

The legacy `Report` remains as an 어댑터.

## Consequences

- 아키텍처 의사결정에서 관련 설정과 실행 흐름을(를) 기준으로 데이터 품질 검증, 워크플로우 자동화, 결과 해석 방법을 설명합니다.
- 아키텍처 의사결정에서 관련 설정과 실행 흐름을(를) 기준으로 데이터 품질 검증, 워크플로우 자동화, 결과 해석 방법을 설명합니다.
- 아키텍처 의사결정에서 관련 설정과 실행 흐름을(를) 기준으로 데이터 품질 검증, 워크플로우 자동화, 결과 해석 방법을 설명합니다.

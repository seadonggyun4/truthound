# Testing and CI Ownership

오케스트레이션 실행에서 Truthound, Orchestration을(를) 기준으로 데이터 품질 검증, 워크플로우 자동화, 결과 해석 방법을 설명합니다.

## Who This Is For

- contributors adding 어댑터 coverage
- 오케스트레이션 실행에서 관련 설정과 실행 흐름을(를) 기준으로 데이터 품질 검증, 워크플로우 자동화, 결과 해석 방법을 설명합니다.
- 오케스트레이션 실행에서 관련 설정과 실행 흐름을(를) 기준으로 데이터 품질 검증, 워크플로우 자동화, 결과 해석 방법을 설명합니다.

## When To Use It

오케스트레이션 실행에서 관련 설정과 실행 흐름을(를) 다루는 항목입니다:

- 오케스트레이션 실행에서 `packages/<adapter>/tests`, `tests/<adapter>`을(를) 기준으로 데이터 품질 검증, 워크플로우 자동화, 결과 해석 방법을 설명합니다.
- 오케스트레이션 실행에서 관련 설정과 실행 흐름을(를) 기준으로 데이터 품질 검증, 워크플로우 자동화, 결과 해석 방법을 설명합니다.
- 오케스트레이션 실행에서 관련 설정과 실행 흐름을(를) 기준으로 데이터 품질 검증, 워크플로우 자동화, 결과 해석 방법을 설명합니다.

## Minimal Quickstart

오케스트레이션 실행에서 관련 설정과 실행 흐름을(를) 다루는 항목입니다:

```bash
PYTHONPATH=.:packages/mage/src pytest --import-mode=importlib packages/mage/tests
```

오케스트레이션 실행에서 dbt을(를) 다루는 항목입니다:

```bash
PYTHONPATH=. pytest --import-mode=importlib tests/dbt/test_first_party_suite.py
```
